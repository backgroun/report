# CLAUDE.md

## Existing Technical Rules

# Highest Priority Rule

이 프로젝트는 일반적인 온라인 쇼핑몰이 아니다.

이 웹사이트는 "오프라인 방문 유도형 디지털 카탈로그"이다.

온라인 판매가 목적이 아니며, 고객이 제품 정보를 탐색한 후 실제 매장을 방문하거나 상담을 진행하도록 유도하는 것이 핵심 목표이다.

Claude는 일반적인 이커머스 패턴(장바구니, 결제, 체크아웃)을 적용하지 않는다.

모든 제안과 구현은 아래 질문으로 검토한다.

> "이 변경이 실제 매장 방문 가능성을 높이는가?"

그렇지 않다면 더 적절한 대안을 제안한다.

---

# 프로젝트 목적

## Primary Goals

- 제품 탐색 경험 제공
- 브랜드 신뢰 형성
- 오프라인 매장 방문 유도
- 전화 문의 증가
- 카카오톡 상담 유도
- 매장 위치 확인 및 길찾기 유도

## Success Metrics

다음 행동을 전환으로 간주한다.

- 매장 찾기 클릭
- 길찾기 실행
- 전화 문의 클릭
- 카카오톡 상담 클릭
- 제품 문의 제출
- 방문 예약 요청
- 매장 운영시간 확인

온라인 결제 전환율은 KPI가 아니다.

# 절대 구현하지 말 것

다음 기능은 사용자가 명시적으로 요청하지 않는 한 제안하거나 구현하지 않는다.

- 장바구니
- 구매하기 버튼
- 체크아웃
- 결제 시스템
- 결제 연동
- 쿠폰 시스템
- 적립금
- 배송비 계산
- 주문 관리 시스템
- 온라인 주문 플로우

쇼핑몰이라는 이유만으로 일반적인 이커머스 기능을 추가하지 않는다.

# 기술 스택

현재 프로젝트의 기술 스택을 최우선으로 존중한다.

기존 코드베이스를 분석한 후 동일한 패턴을 따른다.

임의로 기술 스택을 변경하지 않는다.

새로운 라이브러리 도입 시 반드시 다음 내용을 설명한다.

- 도입 이유
- 장점
- 단점
- 대체 가능 여부
- 기존 방식 대비 이점

# 작업 원칙

항상 아래 순서를 따른다.

## 1. 요구사항 이해

사용자의 요청을 요약한다.

명확하지 않은 부분은 질문한다.

추측하여 구현하지 않는다.

---

## 2. 영향 범위 분석

다음을 식별한다.

- 수정 대상 파일
- 영향받는 기능
- 잠재적인 부작용
- SEO 영향 여부
- 오프라인 전환 영향 여부

---

## 3. 구현 계획 수립

구현 전에 다음을 제시한다.

- 접근 방법
- 단계별 작업 내용
- 예상 위험 요소
- 테스트 계획

사용자가 "바로 구현해줘"라고 명시하지 않는 한 계획 승인을 기다린다.

---

## 4. 구현

구현 시 다음 원칙을 따른다.

- 최소 변경 원칙
- 기존 아키텍처 유지
- 재사용 가능한 구조 우선
- 중복 코드 방지
- 단순한 해결책 우선

---

## 5. 검증

구현 후 반드시 검토한다.

- 오류 가능성
- 예외 상황
- 모바일 환경
- 접근성
- 성능 영향

---

# 제품 상세 페이지 원칙

제품 상세 페이지는 판매 페이지가 아니라 체험 유도 페이지이다.

필수 요소:

- 제품 이미지 갤러리
- 제품명
- 핵심 특징 요약
- 상세 설명
- 주요 스펙
- 추천 사용 상황
- 관련 제품 추천

CTA 요소:

- 매장에서 직접 체험해보세요
- 가까운 매장 찾기
- 전화 문의하기
- 카카오톡 상담하기
- 제품 문의하기
- 재고 문의하기

금지 요소:

- 구매하기
- 장바구니 담기
- 즉시 구매
- 결제 유도 문구

# 매장 정보 페이지 원칙

매장 정보는 핵심 기능이다.

다음을 포함한다.

- 매장명
- 주소
- 전화번호
- 운영시간
- 지도
- 길찾기 버튼
- 주차 가능 여부
- 매장 사진
- 취급 제품
- 상담 가능 여부

모바일 사용자가 빠르게 행동할 수 있도록 설계한다.

# UX 원칙

Mobile First를 적용한다.

사용자는 이동 중 검색한다고 가정한다.

중요 CTA는 첫 화면에서 보여야 한다.

사용자가 다음 행동까지 도달하는 클릭 수를 최소화한다.

권장 사용자 흐름:

제품 탐색
→ 제품 상세 확인
→ 매장 확인
→ 상담 또는 방문 행동

CTA는 명확하고 반복적으로 노출한다.

---

# SEO 원칙

오프라인 검색 유입을 중요하게 고려한다.

다음을 최적화한다.

- 지역명 + 브랜드명
- 지역명 + 제품명
- 지역명 + 매장명
- 지역명 + 전문점
- 제품 체험 키워드

적용 항목:

- Local SEO 고려
- Schema.org LocalBusiness 구조화 데이터
- Open Graph 메타데이터
- 적절한 title 및 meta description
- 이미지 alt 작성

SEO 변경 시 예상 효과를 설명한다.

# 최종 검토 질문

모든 작업이 끝난 후 반드시 스스로 질문한다.

1. 이 변경이 실제 매장 방문 가능성을 높이는가?
2. 사용자가 더 쉽게 문의할 수 있는가?
3. 모바일 사용자가 행동하기 쉬운가?
4. 불필요하게 이커머스 기능을 추가하지 않았는가?
5. 더 단순한 해결책은 없는가?

위 질문 중 하나라도 "아니오"라면 개선안을 함께 제안한다.
