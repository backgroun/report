WORKUP 웹사이트 상세 스토리보드

작성일: 2026년 6월 10일  |  workup-young83.vercel.app
본 문서는 WORKUP 공식 웹사이트의 전체 페이지 구성, 섹션별 내용, 사용자 흐름을 정리한 내부 보고용 스토리보드입니다.

01 · 사이트 개요

사이트 목적

온라인 판매가 아닌 "오프라인 매장 방문 유도형 디지털 카탈로그"입니다. 고객이 제품을 탐색한 후 실제 매장을 방문하거나 전화·카카오 상담으로 이어지도록 설계되었습니다.

전환 목표 (KPI)

  • 매장 찾기 클릭
  • 길찾기 실행
  • 전화 문의 클릭
  • 카카오톡 상담 클릭
  • 매장 운영시간 확인
  • 피팅 리스트 저장

구성 페이지

  • 홈 (Home)
  • Products (상품 목록/상세)
  • Store (매장 찾기)
  • Story (브랜드 스토리)
  • People (고객 이야기)
  • Field Test (현장 검증)
  • 피팅 리스트

방문 준비
핵심 개념
장바구니 ≠ 피팅 리스트
  • 온라인 결제 기능 없음
  • 매장 방문 시 직원에게 보여주는 목록
  • 사이즈·컬러 미리 선택 가능
Section 1
매장 방문 안내
  • "목록을 직원에게 보여주세요"
  • 재고·사이즈 즉시 확인 안내
  • 가까운 매장 찾기 → Store 이동
Section 2
저장된 상품 목록
  • 상품명 / 사이즈 / 컬러
  • 개별 삭제 / 전체 비우기
Section 3
하단 CTA
  • 매장 찾아서 입어보기 → Store
  • 제품 더 보기 → Products

주요 CTA

매장 찾아서 입어보기 → 제품 더 보기
페이지 목적: 온라인 결제가 아닌, 관심 상품을 정리해 매장 방문을 더 편리하게 만드는 페이지입니다. 최종 목적지는 항상 Store 페이지입니다.
